# Docker Notes

## Why This Repository Exists

I have a habit of documenting concepts that I struggle with or believe I might forget in the future.

While learning Docker and deployment, I found myself repeatedly revisiting the same topics:

* Docker Images vs Containers
* Dockerfiles
* Docker Compose
* Environment Variables
* Docker Hub
* GitHub Actions
* CI/CD
* Deployment Pipelines

Most tutorials explained *how* to do things, but rarely explained *why* they worked the way they did.

This repository is my attempt to build a personal knowledge base that explains Docker and deployment concepts from first principles.

The goal is simple:

```text
Learn once.
Document properly.
Never relearn from scratch.
```

---

## Why I Document Things

Whenever I encounter a topic that feels confusing, I create documentation for it in my own words.

This helps me:

* Understand concepts more deeply
* Create a future reference for myself
* Reduce the need to repeatedly search the same topic
* Build a structured knowledge base over time

Some people use bookmarks.

Some people save YouTube videos.

I prefer writing documentation.

---

## What This Repository Covers

This repository focuses on Docker and modern deployment workflows.

Topics include:

* Why Docker Exists
* Docker Images, Containers and Dockerfiles
* Common Docker Commands
* Docker Compose
* Docker Networks and Volumes
* Docker for Spring Boot
* Docker Hub and Registries
* CI/CD with GitHub Actions
* Complete Deployment Flow

The documents are written primarily from the perspective of a backend developer working with Spring Boot applications.
